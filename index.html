Security Knowledge

Master the fundamentals of password security and cryptography

Cryptographic Hashing

Passwords are never stored in plain text. They're transformed using hash functions like SHA-256, bcrypt, or Argon2 into fixed-length strings. These are one-way functions — you can't reverse the hash to get the original password.

password123 → SHA-256 → ef92b778... Password123 → SHA-256 → b2e98ad6...

Even a tiny change produces a completely different hash — this is called the avalanche effect.

Password Entropy

Entropy measures randomness in bits. Higher entropy = harder to crack. It's calculated as:

E = L × log₂(R)

Where L = password length and R = size of the character pool (charset). A password using lowercase + uppercase + numbers + symbols has R = 95.

< 28 bitsVery Weak
28-35 bitsWeak
36-59 bitsFair
60-127 bitsStrong
128+ bitsExcellent

Common Attack Methods

Brute Force

Tries every possible combination. Speed: ~10 billion guesses/sec (GPU)

Dictionary Attack

Uses lists of common words, names, and known passwords

Rainbow Tables

Precomputed hash-to-password lookup tables (defeated by salting)

Credential Stuffing

Reuses leaked username/password pairs across different services
